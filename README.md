# Telegram Warmup Bot (Chat Join Request)

Бот ловит заявки на вступление в закрытый канал и запускает прогрев в личку пользователю. После завершения прогрева автоматически одобряет заявку.

## Как это работает

1. Пользователь подаёт заявку на вступление в закрытый канал
2. Бот получает событие `chat_join_request`
3. Бот пишет пользователю в личку цепочку сообщений с паузами
4. После завершения прогрева бот одобряет заявку — пользователь попадает в канал

## Локальный запуск

```bash
cd bot
npm install
cp .env.example .env
# заполни .env своими значениями
npm start
```

## Настройка

1. Создай бота через [@BotFather](https://t.me/BotFather) — получи `BOT_TOKEN`.
2. Создай закрытый канал (или группу) с **заявками на вступление**:
   - Настройки канала → Тип канала → Частный
   - Включи "Одобрение новых подписчиков" (Approve new members)
3. **Добавь бота в канал как администратора** с правами:
   - "Добавление участников" (чтобы мог одобрять заявки)
4. Узнай `CHANNEL_ID`:
   - Добавь бота [@userinfobot](https://t.me/userinfobot)
   - Перешли ему любое сообщение из канала — он покажет ID

## Редактирование прогрева

Открой `warmup.js` — там массив шагов:
- `text` — текст сообщения (поддерживает HTML: `<b>`, `<i>`, `<a href="">`)
- `delay` — пауза в миллисекундах перед следующим сообщением

## Деплой на Railway

1. Залей папку `bot/` в GitHub-репозиторий
2. На [railway.app](https://railway.app): **New Project → Deploy from GitHub repo**
3. Если бот в подпапке — в настройках задай **Root Directory** = `bot`
4. В **Variables** добавь:
   - `BOT_TOKEN` — токен от BotFather
   - `CHANNEL_ID` — ID канала (например `-1001234567890`)
5. Railway сам выполнит `npm install` и `npm start`

Бот работает в режиме long polling — порт открывать не нужно.

## Кастомизация

**Одобрить заявку сразу (до прогрева):**
В `index.js` раскомментируй строку `await approveJoinRequest(...)` в обработчике `chat_join_request`.

**Не одобрять автоматически:**
Закомментируй вызов `approveJoinRequest` в `.then()` после `runWarmup`.
